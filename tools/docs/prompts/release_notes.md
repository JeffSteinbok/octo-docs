Generate release notes documentation with the following sections:

## Structure

1. **Title** (H1) — e.g. "Release Notes" or "What's New"
2. **Summary** — Two to three sentences summarizing the most important changes
3. **Breaking Changes** — Bullet list of breaking changes, or omit section if none
4. **New Features** — Bullet list of new capabilities added
5. **Bug Fixes** — Bullet list of bugs fixed, if present
6. **Migration Guide** — Only if migration steps are described in the source material

## Constraints
- Keep the tone factual and concise
- Do not invent changes not present in the source
- Omit empty sections
- Format consistently
